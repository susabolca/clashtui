# Mihomo Config Subset

This is the mihomo config subset relevant to clashtui. It is not the full
mihomo manual.

Important top-level fields:

- `mixed-port`: local mixed HTTP/SOCKS listener.
- `port`: local HTTP listener.
- `socks-port`: local SOCKS listener.
- `allow-lan`: whether local listeners accept LAN clients.
- `mode`: `rule`, `global`, or `direct`.
- `external-controller`: controller address used by clashtui.
- `secret`: optional controller secret.
- `listeners`: per-Port Proxy local listeners generated by clashtui.

DNS fields:

- `dns.enable`
- `dns.listen`
- `dns.enhanced-mode`
- `dns.fake-ip-range`
- `dns.fake-ip-filter`
- `dns.fake-ip-filter-mode`
- `dns.default-nameserver`
- `dns.nameserver`
- `dns.nameserver-policy`
- `dns.fallback`
- `dns.fallback-filter`
- `dns.proxy-server-nameserver`
- `dns.direct-nameserver`
- `dns.direct-nameserver-follow-policy`
- `dns.respect-rules`
- `dns.use-hosts`
- `dns.use-system-hosts`

TUN fields:

- `tun.enable`
- `tun.stack`
- `tun.device`
- `tun.auto-route`
- `tun.auto-redirect`
- `tun.auto-detect-interface`
- `tun.dns-hijack`
- `tun.strict-route`
- `tun.mtu`
- `tun.route-exclude-address`

Profile fields usually come from subscriptions:

- `proxies`
- `proxy-groups`
- `proxy-providers`
- `rules`
- `rule-providers`

The assistant must modify clashtui AppConfig, not generated mihomo YAML.
